בסיסי נתונים

מה זה בסיס נתונים?

(5 דקות)

  • מקום לשמור נתוני האפליקציה
  • לא באחריותך
  • מאפשר סידור של הנתונים
  • לכל סביבה יש:
    • מנוע (שרת) שאיתו נגישים לנתונים
    • לפחות בסיס הנתונים אחד (כלומר מארז הנתונים)
  • דואג לביצועים ע”י מתמטיקה מתקדמת
  • לא ממש “דבר” אחד אלא קטגוריה

למה להשתמש בה?

(5 דקות)

  • להוציא אחריות על נתונים מאפליקציות
  • להפריד בין נתונים לבין פונקציונליות אפליקטיבית
  • לשפר ביצועים של אפליקציות

SQL ו-NoSQL

מה זה SQL?

(5 דקות)

  • Structured Query Language
  • שפה בה נגישים לבסיס נתונים רלציוני (טבלאי-מקושרת)
  • Declarative
  • טבלאות מקושרים
  • הרחבות לפי בסיס נתונים ספציפי
  • אפשר להשתמש בעמוד NoSQL בבסיס נתונים SQL

מה זה NoSQL?

(5 דקות)

  • Non-Structured Query Language
  • כלומר כל בסיס נתונים שלא מדבר SQL
  • דוגמאות:
    • Document (הסבר בהמשיך)
    • Key-Value (כמו Redis)
    • Graph

BSON

(10 דקות)

  • פורמט בינארי דומה ל-JSON

  • סוגי נתונים נוספים:

    • ObjectID
    • Double
    • 32-bit integer (int)
    • 64-bit integer (long)

ObjectID

(10 דקות)

  • טיפוס מזהה של מסמכי MongoDB
  • מורכב מ-12 בתים:
    • 4 בתים עם הזמן (timestamp)
    • 5 בתים אקראיים
    • 3 בתים צבורים ממספר אקראי
  • תפקידו לדאוג לשוני בין מסמכים הנוצרים במחשים שונים
  • מאפשר scaling של cluster (כלומר horizontal scaling)

Collection

(5 דקות)

  • Collection הוא מארז של מסמכים
  • מקביל ל-table בבסיס נתונים רלציוני

Cursor

(10 דקות)

  • כמעט כל שאילת שמבצעים (ב-driver) מחזיר cursor
  • הוא iterable (כלומר אפשר לעבור עליו בלולא)
  • המסמכים עצמם רק הוחזרו ברגע שעוברים על cursor
  • אפשר לבקש את רק גודל השיאלת (שיטת size)
  • אפשר לסדר מסמכים ב-cursor (שיטת sort)
  • אפשר גם לדלג (שיטת skip) אבל לא כדאי

ולידציה (רבע שעה)

  • ל-MongoDB אין schema
  • יש ולידציה
  • אפשר לעשות ולידציה מול JSON schema (draft 4)

דוגמאת schema

  const gradesSchema = {
    $jsonSchema: {
      required: [ "name", "major", "gpa" ],
      properties: {
          name: {
            bsonType: "string",
            description: "must be a string and is required"
          }
      }
    }
  }

דוגמאת שימוש ב-schema

// Create grades collection with validation
db.createCollection( "grades", { validator: { $jsonSchema: gradesSchema } } )
// Or modify an existing collection to add the validation
db.runCommand( { collMod: "grades", validator:{ $jsonSchema: gradesSchema } } )

אופציות לולידציה

  • אפשר להריץ ולידציה רק על מסמכים שכבר עומדים בולידציה
  • אפשר רק להתריע בלוגים על כישלון בולידציה ולא למנוע פועלות

Aggregation

  • דרך לאסוף/לחבר בין מסמכים
  • מתחיל עם collection מסויים
  • הכי קל להבין מדוגמאות

$addFields (5 mins)

  •   // Scores collection
  {
    _id: 1,
    student: "Maya",
    homework: [ 10, 5, 10 ],
    quiz: [ 10, 8 ],
    extraCredit: 0
  }
  // Aggregation pipeline
  db.scores.aggregate( [
    {
      $addFields: {
        totalHomework: { $sum: "$homework" } ,
        totalQuiz: { $sum: "$quiz" }
      }
    },
    {
      $addFields: { totalScore:
        { $add: [ "$totalHomework", "$totalQuiz", "$extraCredit" ] } }
    }
  ])

$addFields output

  •   {
    "_id" : 1,
    "student" : "Maya",
    "homework" : [ 10, 5, 10 ],
    "quiz" : [ 10, 8 ],
    "extraCredit" : 0,
    "totalHomework" : 25,
    "totalQuiz" : 18,
    "totalScore" : 43
  }

$lookup (quarter hour)

  • אין Foreign Keys ב-MongoDB
  • $lookup מאשפר חיבור בין collections בדומה ל-JOIN ב-SQL (מסוג LEFT OUTER)
  • עדיף לא לחבר אלא להכניס נתונים לאותם מסמכים

$lookup example

  •   // Order collection
  { "_id" : 1, "item" : "almonds", "price" : 12, "quantity" : 2 }
  { "_id" : 3  }
  // Inventory collection
  { "_id" : 1, "sku" : "almonds", "description": "product 1", "instock" : 120 }
  { "_id" : 2, "sku" : "bread", "description": "product 2", "instock" : 80 }
  { "_id" : 5, "sku": null, "description": "Incomplete" }
  { "_id" : 6 }
  // Aggregate pipeline
  db.orders.aggregate([
    {
      $lookup:
        {
          from: "inventory",
          localField: "item",
          foreignField: "sku",
          as: "inventory_docs"
        }
    }
  ])

$lookup output

  •   // Output
  {
    "_id" : 1,
    "item" : "almonds",
    "price" : 12,
    "quantity" : 2,
    "inventory_docs" : [
        { "_id" : 1, "sku" : "almonds", "description" : "product 1", "instock" : 120 }
    ]
  }
  {
    "_id" : 3,
    "inventory_docs" : [
        { "_id" : 5, "sku" : null, "description" : "Incomplete" },
        { "_id" : 6 }
    ]
  }
  • שימו לב from collection נדרש להיות לא sharded

Isolation

(5 דקות)

מיקבול של transactions יביא את בסיס הנתונים לאותו מצב שבו הוא יגיע עם הם יופעלו אחד-אחרי-השני

Durability

(5 דקות)

Transactions שנשמר ישומר כמו שהוא גם במידע וחומרה קורסת (למשל: קריסת חשמל)

תרגיל (שעה וחצי)

התחלה (~חצי שעה)

  1. הכנסו לחשבון ה-OpenShift שלכם והריצו את ה-CLI של MongoDB.
  2. תצור collection חדש עבור שמירת חחליות. החלליות מכילות עד 6 היגויום (מספר שרורתי) ועד 2 שיטות דחף (כדי לא לבזבז משקל). המסמכים. משקל חללית לא תעלה מעל 200,000 קילוגרם. עליך ליצור ולידציה עבור זה.

יצירת נתונים (~רבע שעה)

  1. תצור שלוש חלליות:
    1. ממונע ע”י מנוע עם דלק נוזלי ואלטרומגנטי עם 4 היגויים עם משקל סה”כ של 100,000 קילוגרם.
    2. ממונע ע”י מנוע עם דלק נוזלי ודלק מוצק עם 6 היגויים עם משקל סה”כ של 200,000 קילוגרם.
    3. ממונע ע”י מנוע עם דלק מוצק ואלטרומגנטי עם 5 היגויים עם משקל סה”כ של 150,000 קילוגרם.
    4. ממונע ע”י מנוע עם דלק מוצק ודלק נוזלי עם 7 היגויים עם משקל סה”כ של 250,000 קילוגרם. תברר כי מסמך זה לא הוכנס בהצלחה.

הארכת נתונים (~רבע שעה)

  1. תארוך את החללית עם משקל 150,000 קילוגרם (ללא שימוש בשדות אחרות) כדי להחליף לו את המנועים שלו למנוע אחד מבוסס anti-matter. משקלו יורד ל20,000 קילוגרם.
  2. תמחוק את החללית עם 5 היגויים.

אגרגציה (~חצי שעה)

  1. תיצור סכום של משקלם של כל החלליות ע”י אגרגציה.

אם נשאר לכם זמן: Transactions

  1. תארוך את שני החלליות הקיימות לשדרג אותם עם חומרים חדשים כך שמשקלם יורד ב10%.
